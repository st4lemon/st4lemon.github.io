Projects

Paxos Blockchain | C++ | CS171

Distributed blockchain-based banking system optimized for consistency

  • Implemented the Paxos consensus algorithm to achieve agreement with 5 nodes on the network despite possible crash failures
  • Designed a peer-to-peer network using TCP sockets to allow nodes to communicate
  • Developed a command-line interface for users to check balances and make transfers
  • Ensured durability by persisting blockchain to disk and sending synchronization messages

KOS Operating System | C, Linux | CS170

Operating system simulated on a MIPS R3000 processor

  • Developed code to perform Linux system calls including console I/O and fork/exec/wait
  • Utilized Kthreads and virtual memory to perform multithreading, allowing concurrent execution of 8 processes at once
  • Implemented file descriptors and pipes

Image Stitcher | Python, OpenCV, NumPy | CS181

Panoramic image stitcher using traditional homography methods

  • Implemented RANSAC method to compute homography matrices
  • Utilized SIFT to compute matches between adjacent images

UCSB Dining Commons Menu Item Reviews | Java, React, Node.js, Spring Boot, Dokku, PostgreSQL | CS156

Web application allowing students to review menu items served at UCSB dining halls

  • Created REST API endpoints for creating, retrieving, updating, and deleting reviews
  • Constructed unit and integration tests for backend and frontend components

Cflat Compiler | C++, x86-64 | CS160

Built a compiler from scratch for a simplified C-style programming language.

  • Implemented a longest-match algorithm for lexing and tokenizing programs
  • Created a parser using an LL(1) context-free grammar to translate tokens into an abstract syntax tree
  • Wrote code to translate a lower-intermediate representation into x86-64 assembly code
  • Large project: over 5000 lines of code

Pipelined MIPS CPU Design | Python, PyRTL | CS154

CPU design supporting MIPS architecture

  • Split instruction execution into 5 pipelined cycles
  • Implemented forwarding and stalling to improve performance
  • Created a hierarchical page table and round-robin replacement cache for fast emmory access

LRU-K Replacement Buffer | C/C++, WSL, Git | CS174B

Database buffer using a k-th Least Recently Used replacement strategy, based upon BusTub from Carnegie Mellon University.

  • Constructed a buffer pool manager to access data in memory, and a replacer to determine which frames to replace
  • Ensured functions were thread-safe with page locking techniques
  • Large codebase: referenced the implementation of the disk manager and pages to inform coding decisions
